Overview

This project is a movie application called “myFlix”, providing users with access to information about different movies, directors, and genres, where users can sign up, update their personal information, and create a list of their favourite movies.

Purpose & Context

This is a personal project I built as a part of my Full-Stack Web Development course at CareerFoundry and meant to acquire skills in both the front-end and back-end of an application, creating databases and developing with React.

Objective

Provide users with information about a pool of films, and an ability to save their favourites, in a seamless and bug-free app.

Duration

12 weeks in total (backend: 3 weeks, frontend: 9 weeks)

Credits

Tutor: Blaise Bakundukize
Mentor: Nizar Triki

Tech Stack & Tools

Backend:
Express, Morgan, bodyParser, mongoose, CORS, Passport, bcrypt, MongoDB, Postman, Heroku, JSDoc

Frontend:
JavaScript, React, React Redux, React Bootstrap, Parcel, Axios

Approach

Developing the Backend part

Creating a RESTful API

For the back-end part I listed out all the app's functionalities that would need to interact with the movies' and user database, and then I translated these functionalities into HTML endpoints to build out the structure of the Express API. This preparation step kept the initial coding efficient, helping me know exactly where to start with my project.

I wrote a non-relational database in MongoDB and updated my API so information from the database can be retrieved and updated using HTTP methods like GET or POST.

User authentication

An important part of this step was adding layers of security to protect the data, like JWT authentication which requires users to sign in to retrieve and write data, and password hashing that prevents anyone (even the owner of the database) from viewing a user's password.

All testing on the service-side was done in Postman and accompanied by a documentation, listing the API endpoints with it's required data input and responses.

Creating the Frontend part

Introduction to React

After completing the API I started to build the front-end part of the “myFlix” app, allowing users to make requests and receiving responses from the server-side.

It is a single-page app with several views, created with React and Redux, which meant I needed to familiarise myself with React's component structure and components' data storage to display the right information based on the user interaction, e.g. a user who is not yet registered or logged out, should not see the a list of movies or profile information.

Building the interface

I began building the movies' view, allowing users to see a list of all movies and detail information of movie, director and genre, followed by the views around the user and adding the functionality to update or delete a user's profile.

Building the app's interface and views was sped up my using React-Bootstrap, a UI library which also ensured design consistency, but was customised during developing the views.

Retrospective

Challenges

The movie app was the most challenging project within the Full-Stack Web Developement course. While I thought the server-side part will be the most difficult one, I underestimated the time investment and development scope of the client side, especially building in all functionalities.

During the build process of the client-side I spent a lot of time in customising the React-Bootstrap theme to give my movie app an individual look, which meant less time spent on building in the user functionality. React was a complete new concept to me, understanding the data flows and how the components work in conjunction is something I did underestimate and not fully comprehend first, which led to a lot of errors and bugs in building in the user's functions and components (e.g user profile was not visible anymore).

Realising that I need to fully grasp React in order to finish the app bug-free, made me stop working on the app and properly investing the time for reading it's official documentation, watching YouTube tutorials and additionally reaching out to my tutor and mentor for help, in order to properly continue the build process. Thankfully with my mentor's advice on how to best proceed and my tutor's help, I could fix the bugs and implement all the functionalities properly.

Conclusion & Key takeaways

The movie app was only the second full web app I've developed within the Full-Stack web-developement course. Since developing the app, I have continued to learn more about developing practices, especially the test driven development approach. This helped to primarily focus on building the functionalities first and preventing errors in the development process of the following project, an app that also used React for the client-side.

Main takeaways for future projects are:

  • Focus on understanding the concept (here: React and Redux) first
  • Map out the most essential user information (states) and information flow (state-flow) towards the components
  • Test every single unit features/function in itself within the concept
  • Apply design after error free functionalities are implemented
